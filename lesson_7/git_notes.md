# GIT

## Основные команды

```
git init - создать репозиторий
git add <filename> - добавить файл в индекс
git commit -m 'message' - закоммитить изменения
git status - показать состояние файлов
git push - выгрузка локальных коммитов в удаленный репозиторий (может быть флаг -f, который перезапишет изменения)
git pull - подлить данные из удаленного репозитория
git log - просмотреть список коммитов
```

## Работа с файлами и коммитами

```
git commit --amend -m "comment" - исправить последний коммит
git show hash_commit - показать детали коммита
git diff - показать разницу между предыдущим коммитом и текущим состоянием
git diff 09bd8cc..ba25c0ff - посмотреть разницу между двумя коммитами
git checkout 09bd8cc1 hello.txt - вернуть файл к состоянию коммита 09bd8cc1
git restore --staged <file> - удалить изменения файла из отслеживаемых
git reset <file> - изъять файл, который по ошибке добавили в отслеживаемые
```

## Отмена коммитов

```
git revert HEAD - отменить последний коммит, при этом создается коммит, который отменяет изменения
git revert <commit_hash> - откатиться до коммита с определенным SHA
git reset --soft <commit_hash> - отменит все до коммита <commit_hash>, при этом изменения, которые были после, окажутся в индексе
git reset --mixed <commit_hash> - отменит все до коммита <commit_hash>, при этом изменения, которые были после, окажутся not staged
git reset --hard <commit_hash>- отменит все до коммита <commit_hash>, при этом изменения, которые были после, исчезнут
```

## Работа с конфигурацией

Эти настройки включены в коммиты

```
git config --list - показать текущие настройки
git config --global user.name "Ваше Имя" - установить глобальную настройку с именем
git config --global user.email "Ваше email@gmail.com" - установить глобальную настройку с почтой
```

## Работа с удаленным репозиторием

```
git remote add origin <url> - добавить удаленный репозиторий
git push <remote> <branch> - отправить изменения в удаленный репозиторий
git pull - забрать изменения из удаленного в локальный репозиторий
git clone <url> - склонировать репозиторий себе локально
git fetch - обновляет состояние локального репозитория до состояния удаленного, безопаснее чем pull, т к оставляет текущую работу нетронутой
```

## Работа с ветками

[статья про merge и rebase](https://habr.com/ru/articles/432420/)

```
git branch - показать список веток
git checkout -b <имя_новой_ветки> - создать ветку и переключиться на нее
git checkout <имя_существующей_ветки> - переключиться на существующую ветку
git branch -d <имя_ветки> - удалить ветку
git merge <имя_ветки> - сделать слияние текущей ветки с <имя_ветки> (делается одним коммитом)
git rebase master - переместить коммиты текущей ветки в мастер
```
